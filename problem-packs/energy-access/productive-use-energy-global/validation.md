# Validation

## Validation Layers

| Layer           | Gate                                                                 |
| --------------- | -------------------------------------------------------------------- |
| Structure       | `pnpm validate` passes schemas and problem-pack completeness checks  |
| Evidence        | Every claim has a dated evidence record                              |
| Reproducibility | Quantitative artifacts include rerunnable steps                      |
| Review          | Required reviewer roles approve or request changes                   |
| Replication     | Safety-sensitive or quantitative claims are independently reproduced |

## Baseline Requirements

Sub-national productive-use analyses must document:

- Productive-use data source (mini-grid operator, enterprise survey, value-chain study) with data granularity assessment.
- Agricultural value-chain specificity — document which value chains (milling, drying, cold storage, irrigation, processing) are analyzed.
- Access-vs-use distinction — explicitly state whether data captures electricity connection or actual productive-use application.
- Context-dependency documentation — document market infrastructure, human capital, and equipment-affordability context.
- Mini-grid customer data limitations — document what share of connections have verified productive-use applications.

## Productive-Use Mapping Requirements

No productive-use gap claim may be called actionable until it includes:

- Value-chain-specific analysis — document electricity-demand requirements and adoption barriers for each value chain.
- Data-granularity assessment — document whether data captures connection counts or actual productive-use applications.
- Context-dependency — document how market access, human capital, and equipment affordability affect productive-use potential.
- Comparison with at least one published productive-use analysis for a similar LMIC context.
- Sensitivity analysis showing how productive-use estimates change under alternative adoption assumptions.

## Income-Impact Requirements

No income-impact claim may be merged without:

- Context-dependency documentation — explicitly state that electrification in isolated communities without market access may have near-zero productive-use impact.
- Confound controls for market infrastructure, human capital, and baseline income.
- Access-vs-use distinction — explicitly state that electricity access does not equal verified productive use.
- Non-use case documentation (not for income-gain projections without observed productive-use data, not for overgeneralizing context-dependent results).
