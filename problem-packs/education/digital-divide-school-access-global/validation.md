# Validation

## Validation Layers

| Layer           | Gate                                                                 |
| --------------- | -------------------------------------------------------------------- |
| Structure       | `pnpm validate` passes schemas and problem-pack completeness checks  |
| Evidence        | Every claim has a dated evidence record                              |
| Reproducibility | Quantitative artifacts include rerunnable steps                      |
| Review          | Required reviewer roles approve or request changes                   |
| Replication     | Safety-sensitive or quantitative claims are independently reproduced |

## Baseline Requirements

Sub-national school-connectivity analyses must document:

- Connectivity-data source (Giga, school census, mobile-network proxy) with classification-methodology documentation.
- Data currency — explicitly document when data was collected and whether it reflects current conditions.
- Cross-validation status between Giga satellite data and school census data where both exist.
- Electricity-access context — document whether connected schools have reliable electricity.
- Connectivity-definition documentation — explicitly state what 'connected' means in each data source.

## Connectivity-Mapping Requirements

No connectivity-gap claim may be called actionable until it includes:

- Classification-methodology documentation — explicitly state whether satellite-derived, survey-based, or network-coverage proxy.
- Data-currency assessment — document how old the data is and whether infrastructure changes since collection are likely.
- Cross-validation against at least one independent data source where available.
- Connectivity-quality caveat — document that classification does not capture bandwidth, reliability, or actual educational usage.
- Comparison with at least one published connectivity-gap analysis for a similar low-income country context.

## Digital-Learning Requirements

No digital-learning-effectiveness claim based on connectivity data may be merged without:

- Connectivity-vs-usage distinction — explicitly state that connectivity existence does not equal verified educational use.
- Complementary-factor documentation — document electricity, device, teacher digital-literacy, and content availability.
- Quality-dimension documentation — document bandwidth and reliability where data exists.
- Non-use case documentation (not for technology procurement without complementary investments, not for presenting connectivity maps as digital-readiness assessments).
