# Validation

## Validation Layers

| Layer           | Gate                                                                 |
| --------------- | -------------------------------------------------------------------- |
| Structure       | `pnpm validate` passes schemas and problem-pack completeness checks  |
| Evidence        | Every claim has a dated evidence record                              |
| Reproducibility | Quantitative artifacts include rerunnable steps                      |
| Review          | Required reviewer roles approve or request changes                   |
| Replication     | Safety-sensitive or quantitative claims are independently reproduced |

## Baseline Requirements

Sub-national skills-training analyses must document:

- Training-provider data source and fragmentation assessment — explicitly document coverage across government, NGO, and private operators.
- Employment-outcome tracking rate — document what share of training programs in the analyzed country track graduate employment.
- Labor-force survey methodology and youth-employment definitions used.
- Informal-sector measurement methodology — document whether and how informal employment is captured in outcome data.
- Skills-mismatch indicator source and methodology.

## Training-Availability Requirements

No training-availability claim may be called actionable until it includes:

- Provider-registry data completeness — document which provider types are captured and which are absent.
- Coverage-gap documentation — explicitly state where provider data is unavailable.
- Training-type differentiation — document which skill categories (vocational, digital, agricultural, soft skills) are mapped.
- Comparison with at least one published training-system assessment for a similar LMIC context.

## Employment-Outcome Requirements

No training-effectiveness claim may be merged without:

- Outcome-tracking rate documentation — explicitly state what share of programs in the analyzed context track graduate employment.
- Enrollment-vs-impact distinction — explicitly state that training enrollment does not equal verified employment impact.
- Heterogeneity documentation — present effect estimates with program-design context (employer-linked vs. standalone).
- Informal-sector measurement gap — document whether informal employment and self-employment are captured in outcome data.
- Non-use case documentation (not for program expansion without outcome evidence, not for presenting enrollment as effectiveness, not for ignoring program-design heterogeneity).
