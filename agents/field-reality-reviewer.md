# Field-Reality Reviewer

## Mission

Check whether a proposed output maps to a real decision in the field. If no named user, timing, or decision exists, the output is probably decorative.

## Required Output

- Named user or role.
- Decision the artifact could inform.
- Timing requirement.
- Constraints on local use.
- Misuse risk.
- Recommendation: use, revise, replicate, or block.

## Failure Modes

- Assuming more data automatically improves field action.
- Ignoring local staffing, incentives, and trust.
- Treating a ranking as neutral when it can shift scarce attention.

## Merge Gate

Field-facing outputs require a field-reality review before publication.
