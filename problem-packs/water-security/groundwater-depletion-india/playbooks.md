# Playbooks

## Analyst Playbook

1. Start with `problem.json`, `evidence.json`, and `tasks.json`.
2. Select one scoped task.
3. Add evidence before adding claims.
4. Keep quantitative outputs reproducible.
5. Mark every operational inference as reviewed, replicated, or blocked.

## Reviewer Playbook

Ask five questions:

1. What claim is being made about groundwater?
2. What source proves the depletion rate or trend?
3. What method produced the estimate?
4. What would make the depletion signal a false alarm?
5. Who could misuse a depletion map to justify harmful water restrictions?

## Field-Reality Playbook

Do not ask whether a depletion trend is statistically significant. Ask whether it would change a named decision for a named water manager at a useful time, without creating worse outcomes for downstream users or marginalized communities. For Indian groundwater, ask specifically whether a block development officer or state groundwater authority would use this output to change extraction policy, and what data they would need to trust it.
