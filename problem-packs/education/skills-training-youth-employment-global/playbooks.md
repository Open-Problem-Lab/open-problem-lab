# Playbooks

## Analyst Playbook

1. Start with `problem.json`, `evidence.json`, and `tasks.json`.
2. Select one scoped task.
3. Add evidence before adding claims.
4. For training-availability mapping: always document provider-registry fragmentation — government, NGO, and private operators are rarely captured in a single dataset. Document coverage gaps where provider data is absent.
5. For employment-outcome analysis: always document outcome-tracking rates — fewer than 10 percent of programs track graduate employment in most LMICs. Document informal-sector measurement gaps.
6. For effectiveness claims: always present pooled effect estimates with heterogeneity documentation — employer-linked programs show 8-12 percentage point gains while standalone programs show near-zero effects.
7. Mark every operational inference as reviewed, replicated, or blocked.

## Reviewer Playbook

Ask five questions:

1. What claim is being made about youth skills-training availability or employment effectiveness?
2. What source proves it — and is it a provider registry, a labor-force survey, or a program evaluation?
3. What method translates training availability into employment-outcome claims, and what is the outcome-tracking rate?
4. What would make it false — particularly if provider data is fragmented, if outcome tracking is below 10 percent, if informal-sector employment is not captured, or if pooled effect estimates mask heterogeneity?
5. Who could misuse it — particularly to expand programs without outcome evidence, to present enrollment as effectiveness, or to ignore program-design heterogeneity?

## Field-Reality Playbook

Do not ask whether a training-availability map is impressive. Ask whether it would change a named decision for a named user at a useful time, without creating worse incentives. For youth skills training in LMICs, ask specifically:

- Would a Kenya National Industrial Training Authority director use this to target training-program expansion, or would it be treated as evidence of effectiveness when outcome data is absent for 90+ percent of programs?
- Would an employer organization use this to identify skills-supply gaps, or would it ignore that training enrollment does not equal verified skill acquisition?
- Would a labor-ministry program designer use this to calibrate program design, or would pooled effect estimates mask the heterogeneity that determines whether a specific program actually works?

The answer must include explicit non-use cases before any field-facing output is published.
