# Playbooks

## Analyst Playbook

1. Start with `problem.json`, `evidence.json`, and `tasks.json`.
2. Select one scoped task.
3. Add evidence before adding claims.
4. For deforestation mapping: always document GMW resolution limitations for narrow fringing mangroves, pre-1996 baseline uncertainty, and driver-classification methodology with ground-verification status.
5. For protection-loss analysis: always document the attenuation-relationship source, bathymetry data quality, and the non-linear relationship between mangrove width and protection.
6. For valuation: always present values as probabilistic expected annual avoided damages, not as actual damage in any specific event. Document damage-function assumptions for informal coastal settlements.
7. Mark every operational inference as reviewed, replicated, or blocked.

## Reviewer Playbook

Ask five questions:

1. What claim is being made about mangrove deforestation or coastal-protection loss?
2. What source proves it — and is it satellite deforestation data, storm-surge modeling, or a protection valuation?
3. What method translates deforestation pixels into protection-loss estimates, and what are the attenuation-relationship and bathymetry-data assumptions?
4. What would make it false — particularly if GMW misses narrow mangroves, if bathymetry data is inadequate, or if attenuation relationships don't transfer to the analyzed coastline?
5. Who could misuse it — particularly to justify aquaculture bans without livelihood alternatives, to displace coastal communities, or to overstate protection values for carbon-credit schemes?

## Field-Reality Playbook

Do not ask whether a protection-loss valuation is impressive. Ask whether it would change a named decision for a named user at a useful time, without creating worse incentives. For Southeast Asian mangroves, ask specifically:

- Would an Indonesian Ministry of Marine Affairs officer use this to prioritize mangrove restoration sites, or would it be used to justify aquaculture bans that destroy livelihoods without providing alternatives?
- Would a coastal-zone management authority use this to set development setbacks, or would it ignore the economic context of coastal land conversion?
- Would a mangrove-restoration practitioner use this to select sites with highest protection return, or would it create false security about restored-mangrove protection equivalent to natural mangroves?

The answer must include explicit non-use cases before any field-facing output is published.
