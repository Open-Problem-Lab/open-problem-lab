# Validation

## Validation Layers

| Layer           | Gate                                                                 |
| --------------- | -------------------------------------------------------------------- |
| Structure       | `pnpm validate` passes schemas and problem-pack completeness checks  |
| Evidence        | Every claim has a dated evidence record                              |
| Reproducibility | Quantitative artifacts include rerunnable steps                      |
| Review          | Required reviewer roles approve or request changes                   |
| Replication     | Safety-sensitive or quantitative claims are independently reproduced |

## Baseline Requirements

Coastal exposure baselines must document:

- SLR projection source, scenario, and timeframe.
- Elevation data source and vertical accuracy.
- Inundation methodology including tidal datum.
- Population data source and year.
- Uncertainty bounds from vertical accuracy and population data.
- Storm-surge and wave-runup assumptions.

## Model Or Signal Requirements

No adaptation framework may be called validated until it includes:

- Explicit deep-uncertainty handling methodology.
- Multiple SLR scenarios with ensemble ranges.
- Compound-risk components and interaction assumptions.
- SIDS-specific governance and fiscal constraints.
- Explicit statement that framework is for discussion, not investment allocation.
- Safeguards against forced-relocation or maladaptation interpretations.
- Replication status for all quantitative analyses.
