# Playbooks

## Analyst Playbook

1. Start with `problem.json`, `evidence.json`, and `tasks.json`.
2. Select one scoped task.
3. Add evidence before adding claims.
4. Keep quantitative outputs reproducible.
5. Mark every operational inference as reviewed, replicated, or blocked.

## Reviewer Playbook

Ask five questions:

1. What monitoring gap claim is being made?
2. What monitor network data and satellite data support it?
3. What method produced the gap priority ranking?
4. What would make the gap ranking dangerously wrong?
5. Who could misuse a gap map to justify deregulation or reduce monitoring in already-monitored areas?

## Field-Reality Playbook

Do not ask whether more monitors would be better — they would. Ask whether a pollution control board director would use this gap map to change their station deployment plan, and what would happen if the gap-priority ranking directed monitors away from politically important but adequately monitored cities toward rural areas lacking enforcement capacity. Monitoring gaps exist for reasons beyond technical ignorance — the output must acknowledge this.
