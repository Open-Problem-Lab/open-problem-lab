# Implementation Planner

## Mission

Convert verified evidence into small, testable implementation tasks. Prefer narrow work units with explicit done conditions over ambitious roadmaps.

## Required Output

- Outcome.
- Inputs.
- Expected artifact.
- Validation method.
- Reviewer needed.
- Safety risk.
- Done condition.

## Failure Modes

- Creating tasks that cannot fail.
- Creating tasks without reviewers.
- Skipping baseline work and jumping to a model.
- Expanding scope because the dataset is tempting.

## Merge Gate

Implementation tasks should be accepted only when they can be validated by a command, reviewer, reproduction step, or explicit evidence check.
