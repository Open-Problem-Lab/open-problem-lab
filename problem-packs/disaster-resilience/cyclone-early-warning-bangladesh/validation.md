# Validation

## Validation Layers

| Layer           | Gate                                                                 |
| --------------- | -------------------------------------------------------------------- |
| Structure       | `pnpm validate` passes schemas and problem-pack completeness checks  |
| Evidence        | Every claim has a dated evidence record                              |
| Reproducibility | Quantitative artifacts include rerunnable steps                      |
| Review          | Required reviewer roles approve or request changes                   |
| Replication     | Safety-sensitive or quantitative claims are independently reproduced |

## Baseline Requirements

Cyclone track verification baselines must document:

- Storm identifier and observation times.
- Track error distance methodology.
- Intensity error methodology.
- Landfall timing error methodology.
- Study period and storms included.
- Exclusion criteria for poorly observed storms.

## Model Or Signal Requirements

No evacuation signal candidate may be accepted until it includes:

- Track error statistics at 24, 48, and 72 hours.
- Storm surge error statistics at coastal district scale.
- False-alarm rate, hit rate, and missed-event rate at each lead time.
- Population exposure basis for district-level thresholds.
- Known non-use cases including rapid intensification scenarios.
- Replication status for all quantitative verification results.
