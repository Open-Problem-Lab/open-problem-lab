# Validation

## Validation Layers

| Layer           | Gate                                                                 |
| --------------- | -------------------------------------------------------------------- |
| Structure       | `pnpm validate` passes schemas and problem-pack completeness checks  |
| Evidence        | Every claim has a dated evidence record                              |
| Reproducibility | Quantitative artifacts include rerunnable steps                      |
| Review          | Required reviewer roles approve or request changes                   |
| Replication     | Safety-sensitive or quantitative claims are independently reproduced |

## Baseline Requirements

Sub-national clean energy financing analyses must document:

- Investment-flow data source (IRENA, Bloomberg NEF, CPI) with sub-national granularity assessment.
- Financing-structure barrier documentation — identify which barriers (missing middle, currency risk, transaction costs) limit access in the analyzed context.
- Concessional-finance allocation data — document country-income-group and technology-type breakdowns.
- Energy-access gap data for the target context.
- Volume-vs-access distinction — explicitly state that increasing finance volume alone does not close access gaps.

## Financing-Gap Mapping Requirements

No financing-gap claim may be called actionable until it includes:

- Sub-national granularity — explicitly state whether data is national aggregate or district-level.
- Data-gap documentation — state where sub-national financing data is absent.
- Investment-geography analysis — document how investment concentrates in urban and industrial areas versus rural last-mile contexts.
- Comparison with at least one published financing-gap analysis for a similar low-income country context.

## Financing-Structure Requirements

No financing-structure claim may be merged without:

- Missing-middle barrier documentation — explicitly state project-size thresholds and why they deter investors.
- Concessional-finance allocation bias — document how concessional flows distribute across country income groups and technology types.
- De-risking instrument coverage — document what instruments exist and their reach.
- Non-use case documentation (not for claiming finance volume alone closes access gaps, not for ignoring structural barriers, not for overgeneralizing national statistics to sub-national contexts).
