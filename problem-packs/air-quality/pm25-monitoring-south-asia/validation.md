# Validation

## Validation Layers

| Layer           | Gate                                                                 |
| --------------- | -------------------------------------------------------------------- |
| Structure       | `pnpm validate` passes schemas and problem-pack completeness checks  |
| Evidence        | Every claim has a dated evidence record                              |
| Reproducibility | Quantitative artifacts include rerunnable steps                      |
| Review          | Required reviewer roles approve or request changes                   |
| Replication     | Safety-sensitive or quantitative claims are independently reproduced |

## Baseline Requirements

Monitoring network gap baselines must document:

- Station location source and date.
- Monitoring parameter and measurement methodology.
- Coverage buffer methodology and justification.
- Population data source and year.
- Administrative boundary source and version.

## Model Or Signal Requirements

No gap priority ranking may be called validated until it includes:

- Explicit weighting methodology with formula.
- Sensitivity to alternative weightings.
- Satellite PM2.5 validation against ground monitors in region.
- Statement that satellite estimates in unmonitored areas are unvalidated estimates.
- Known non-use cases for regulatory standard-setting.
- Replication status for all quantitative results.
